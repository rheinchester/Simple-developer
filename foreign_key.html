Foreign key

In a relational database, a foreign key is a column, which links its table with another table. The main table that the link is created from contains a primary key, which links to the foreign key of the second table. This primary key -> foreign key link allows one or more rows of data in one table to link, or relate, to one or more rows of data in another table. Here’ s an example


From the above, we can see two tables, movie table and category table. We can see they have the “category_id” column in common. In the category table, “category_id” serves as the primary key. While in the movies table, “category_id” serves as the foreign key.
