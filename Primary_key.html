Primary key in DBMS

Posted by Jacob Okoro on August 24, 2017

Primary key

A primary key is a column (or columns) in a table that uniquely identifies the rows in that table. For example.
From the above we see that the user_id is the primary key. It is the only column where no item is repeated. In the first_name column, jayson is repeated twice. In the last_name column, Okoro is repeated twice. Same for jokoro@gmail.com in the email. Since these columns aren’t primary keys, repetition is allowed. In many designs, the primary key is an auto-generated number from an identity column.

Unique key

The UNIQUE key ensures that all values in a column are different. Both the UNIQUE and PRIMARY KEY constraints provide a guarantee for uniqueness for a column or set of columns. A PRIMARY KEY constraint automatically has a UNIQUE constraint. However, you can have many UNIQUE constraints per table, but only one PRIMARY KEY constraint per table. Assuming that Staff_id is the primary key, then we may want to place a unique constraint on email to ensure each staff has a unique email so that one person doesn’t have two accounts. Example is seen in some social networking sites where an email cannot have two accounts. Some forums also ensure that the username is unique so that two people wouldn’t share the same identity.
