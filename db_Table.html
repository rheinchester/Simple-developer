Creating a table

According to Wikipedia, A table is a collection of related data held in a structured format within a database. It consists of columns, and rows.
Each row in a table represents a set of related data, and every row in the table has the same structure.
In the context of relational databases, a column is a set of data values, all of a single type, in a table.
For example, in a table that represents names of individuals in a class, a row represents a single student. While a columns represents the student’s attribute like, age, gpa, name, surname, address, etc.
You’ll get a better understanding as we create the table.
Now looking at your left hand corner, you’ll see something similar to the image above. After left-clicking on the database you created (new_dbase), you’ll see this From the above, we intend to store the staff information of a company. We have created 6 columns for this. After clicking on go, you see these empty entries with no information. You can fill them with any suitable information of your choice. For me these are what I entered. I shall enumerate them below
  • Staff_id: Each staff will have their own specific number (as a unique identifier). For an id, we set the type as INT(integer).For the length we set it as 11, meaning the maximum number of digits used to represent this field is 11. We set the default value as none in all because….. You’ll also notice that A_I is ticked. A_I stands for auto-increment. This means that a unique number is generated automatically when a new record is inserted. E.g the first registered staff will have an id of 1. The next registered staff will have an id of 2. This allows staff_id to act as a primary key (Since a unique number is generated).
  • Username: The username of the staff is of the type varchar. This means a mixture of text and numbers could be used. The default value is 255 (the username should not exceed 255 characters long). You could specify the length with any figure of your choice.
  • Sign_up: I recommend using datetime rather than date as type. Datetime is used because we don’t just want the date of sign up, we also want the precise time of sign up -down to the second- along with the date. To select datetime, click on the caret-down button. Hit “d” until you see datetime
  • Email: We leave it as varchar because we want a mix of text and numbers. The default value like username and for the same reason is 255.
  • Bio: we leave the type as text because it can hold greater number of characters than varchar. If you want an greater number of characters, use long text.
  • Email_activation: Whenever anybody signs up, their email is set to zero as the default. Once they click on an email activation link, it switches to a one. This means that for you to have an active account and do stuff on your account you need an email of one. To change to “as defined”, click on the caret-down button- you’ll see “as defined” as an option.
Say you made a mistake and you want to add one more entry, Go to the top of the page, where you see something similar to the above, specify the amount of columns you want to add, click go.
Now we are done, move to the bottom of the page and click on save Here’s our table If you want to view data in the table, click on browse. Here’s what we see when we do so
